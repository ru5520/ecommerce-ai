## 三、项目必答 10 题（标准答案）

### Q1. 用 2 分钟介绍你的项目

→ 用上面 1 分钟版 + 补一句架构：`Gradio UI → api 业务层 → CLIP/FAISS + LLM → CSV/JSON 数据层`

### Q2. CLIP 原理？为什么选它？

> CLIP 用对比学习，把图像和文本编码到同一空间，训练目标是匹配的图文对相似度高、不匹配的相似度低。  
> 选它是因为业务需要**以图搜款 + 文字搜款共用索引**，零样本即可用，不需要再训分类头。

### Q3. FAISS 用的什么索引？为什么？

> `IndexFlatIP`（内积，向量 L2 归一化后等价余弦相似度）。商品约 200 件，精确暴搜 < 1ms，不需要近似索引。embeddings 单独落盘，万级可切 `IndexHNSWFlat`。

### Q4. LLM 在项目里干什么？为什么不让 LLM 直接推荐？

> 两层：① 中文 → 英文 query rewrite（CLIP 英文语料为主）；② 基于**已检索到的真实商品**写导购文案。  
> 不让 LLM 直接推荐是为了**避免幻觉**——商品 id、价格、库存都来自 CSV，LLM 只润色措辞。没 API Key 时整链路降级到关键词检索。

### Q5. 遇到的最大技术挑战？

> **CLIP 冷启动**：201 张图每次重启要 forward 2–3 分钟。  
> 解法：把 `embeddings.npy + meta.json`（CSV mtime/size/count）落盘，启动先校验指纹，命中直接加载。**3 min → 5 s**。  
> （备选：HF Spaces 上 transformers 返回 ModelOutput 不是 Tensor，写了 `_to_tensor` 兼容。）

### Q6. 订单和库存怎么保证正确性？

> 状态机：`pending → paid → shipped → completed`，取消回滚库存。  
> 下单时逐个 `adjust_stock(-qty)`，任一失败则 `rollback` 已扣项；`threading.Lock` + 临时文件 rename 原子写 JSON。

### Q7. 为什么用 CSV 不用数据库？

> 演示项目，CSV 可读可 Excel 编辑；做了 `.bak` 备份、原子 rename、文件锁。  
> **上生产会迁 SQLite/PostgreSQL**，业务逻辑已和 UI 解耦（`api/` 独立，另有 FastAPI）。

### Q8. 怎么评估检索效果？

> 演示项目无线上埋点。生产会做：CTR、加购率、A/B（纯 CLIP vs +LLM 改写）；离线 100 条 query 人工标注算 top-5 NDCG。

### Q9. 下一步怎么优化？

> 优先级：**Chinese-CLIP** 省 LLM 改写 → SQLite 迁移 → 图+文混合 embedding → 真实支付沙箱 → CI/CD + e2e 测试。

### Q10. 这个项目是你独立做的吗？用了 AI 辅助吗？

> 是独立设计、开发、部署。用 Cursor 等工具辅助写样板代码和查文档，**架构决策、性能优化、踩坑修复都是自己做**。能讲清每个模块为什么这样设计。

## 四、Kaggle 项目话术（各 30 秒）

### House Prices（Top 15%）

> 回归任务，核心是**特征工程 + 模型融合**。做了 TotalSF、HouseAge、TotalBath 等组合特征；区分语义缺失（无泳池）和真缺失；训练测试合并 One-Hot 防类别不一致；目标 log1p 变换处理偏态；XGBoost 60% + LightGBM 40% 融合，RMSE 0.12626。

### Store Sales（时序预测）

> 16 天多店销量预测。lag 1/7/14/28 + rolling 均值，**shift(1).rolling() 防泄露**；油价、节假日、门店特征融合；递归预测逐日 append；双方案 50/50 融合；clip 防止负数销量累积。

### Titanic（0.78229）

> 二分类入门。Title、FamilySize、Pclass×Sex、AgeBand 特征；RandomForest；做了特征重要性和误差分析，理解哪些样本预测错了。

**被追问「Kaggle 和工业区别」**：
> Kaggle 是离线固定数据集冲指标；工业要在线延迟、数据漂移、AB 实验、可解释和工程落地。我的电商项目更接近后者。

## 五、计算机基础速记（实习常考）

### Python

- **GIL**：同一进程多线程不能并行 CPU 计算；IO 密集可用多线程，CPU 密集用多进程
- **深拷贝 vs 浅拷贝**：`copy.copy` / `copy.deepcopy`
- **装饰器**：闭包 + 高阶函数，用于日志、鉴权、计时
- **`*args, **kwargs`**：可变 positional / keyword 参数
- **生成器**：`yield`，惰性求值省内存
- **list vs tuple**：list 可变，tuple 不可变、可哈希当 dict key

### 网络 / OS（简答级）

- **TCP vs UDP**：TCP 可靠有序；UDP 快、无连接（视频、DNS）
- **HTTP 状态码**：200 成功，401 未认证，403 无权限，404 不存在，500 服务端错误
- **进程 vs 线程**：进程资源隔离；线程共享地址空间
- **死锁四条件**：互斥、占有且等待、不可抢占、循环等待

### 数据库（若 JD 提到）

- **索引**：B+ 树，加速查询、写变慢
- **事务 ACID**：原子性、一致性、隔离性、持久性
- **LEFT JOIN vs INNER JOIN**

## 六、机器学习 / 深度学习速记

| 概念 | 一句话 |
|------|--------|
| 过拟合 | 训练好测试差 → 正则、Dropout、更多数据、早停 |
| 偏差 vs 方差 | 欠拟合高偏差；过拟合高方差 |
| 交叉验证 | K-fold 估泛化，选超参 |
| 精确率 / 召回率 | Precision = TP/(TP+FP)；Recall = TP/(TP+FN) |
| ROC-AUC | 排序能力，不受阈值影响 |
| 梯度下降 | 沿负梯度更新参数；SGD / Adam |
| BatchNorm | 稳定训练，加速收敛 |
| Attention | QK 算权重，对 V 加权求和 |
| Transformer | Self-Attention + FFN，并行优于 RNN |
| 对比学习 | 正样本靠近、负样本远离（CLIP 核心） |
| Embedding | 离散 id → 稠密向量 |
| RAG | 检索增强生成，先查知识库再让 LLM 答 |
