## 通用投递信注意事项

### ✅ 一定要做

- **每封都改 [方括号] 里的关键词**，不要原样发
- 主题栏写**"岗位 + 姓名 + 学校 + 亮点"**，HR 一眼看见
- 附件**只发 PDF 简历**（不要 Word，避免格式乱）
- **附两个链接**（GitHub + HF Demo），HR 60% 会点开
- **正文 300 字以内**，言简意赅

### ❌ 不要做

- 不要在一封信里堆所有项目（HR 看不完）
- 不要写"我是 XX 大学的，希望贵公司能给我机会"这种空话
- 不要复制网络模板（HR 一眼识破）
- 不要全 Markdown 格式（很多邮件客户端不渲染，发出来一堆 `#`）
- 不要超过 1 屏（手机能滑完最好）

### 📋 投递时勾选项

每封发送前自查：

- [ ] 主题包含岗位名 + 姓名
- [ ] 开头点出 JD 关键词
- [ ] 附了 GitHub + Demo 链接
- [ ] 写了可入职时间 + 城市 + 每周到岗天数
- [ ] 留了手机号 + 邮箱
- [ ] **附件是 PDF**（不是 .md / .docx）
- [ ] 改了 [方括号] 里所有占位符
